5.Spinnaker流水线实践-基础概念与配置

发表于2020-12-18|更新于2021-04-16|Spinnaker
|阅读量:

1.流水线基础感念

应用部署

1.使用Spinnaker的应用程序部署功能来构建和管理连续交付工作流程。由一系列阶段组成。可配置根据事件自动触发。管道消息通知。

2.Stage：描述了一个较高级别的动作的管道将执行线性或并行顺序任务和由阶段的集合。

创建流水线

2.流水线配置

1.执行选项Execution Options

  • 并行触发管道（同时仅允许允许一次）
  • 自动取消在队列中等待的管道
  • 禁用了并发管道执行，那么在下一个执行开始时，等待队列中的管道将被取消。
    • 要将它们保留在队列中，请选中此框。

2.参数化构建

  • Parameter ：可选，如果选中，则此参数将始终显示在管道执行视图中，否则默认情况下将被折叠。

3. 消息通知

选择邮件通知

编辑配置文件

4.自动触发器 Automated Triggers

  • 权限： 当使用自动触发器触发管道时，确定角色是否具有访问受保护的应用程序或账户的权限
  • 要从受保护的应用程序或帐户读取，管道必须至少具有一个对应用程序或帐户具有读取访问权限的角色。
  • 要写入受保护的应用程序或帐户，管道必须至少具有一个对应用程序或帐户具有写访问权的角色。

[^注意：为防止特权提升漏洞，用户必须是此处指定的所有组的成员，才能进行修改和执行管道。]:

类型：Artifactory、Concourse、Cron、Git、Jenkins、Nexus、Pipeline、Plugin、TravisCI、WebHook、Wercker、Docker Registry。

阶段运行选项

阶段运行选项-阶段失败后执行操作

  • 停止整个管道
  • 停止依赖目当前阶段的所有阶段，允许其他的阶段运行。
  • 停止依赖当前阶段的所有阶段，其他阶段完成后标记失败。
  • 忽略失败

阶段运行选项-将执行限制为特定的时间窗口

阶段运行选项-其他

常用的阶段类型

AWS CodeBuild

  • 通过指定项目名称来运行AWS CodeBuild构建。构建使用的源和映像可以来自上游阶段。生成的工件可以注入到管道中，并由下游阶段使用。

Jenkins 触发Jenkins作业

Pipline 运行spinnaker流水线

Wait 等待时间

WebHooK 发送WebHook请求

ManualJudgment 手动判断

  • Instructions: 在进行手动判断时会向用户显示说明。可以包含HTML。
  • Propagate Authentication: 传递身份认证权限，选中后管道具有批准者的权限。
  • Judgment Inputs: 供用户输入的选项。
  • Send Notifications: 发送消息通知，等待判断、判断为继续、判断为停止。

Check Preconditions 检查前置条件

  • 在继续操作之前，请检查前提条件。例如集群是否为特定大小、管道表达式、阶段状态。

Bake(Manifest)

  • 使用诸如Helm之类的模板渲染器生成清单文件（或多文档清单集）

Delete/Deploy/Disable/Enable（Manifest）

  • 资源文件可以从版本控制系统中获取。

7. CICD部分实现

  • 开发人员向Gitlab提交代码，触发JenkinsCI构建。
  • Jenkins构建该分支，生成docker镜像，上传到Harbor镜像仓库。
  • Spinnaker检测到JenkinsCI构建完成后运行CD部署流水线。
